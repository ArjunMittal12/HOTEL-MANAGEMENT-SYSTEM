# Contributing to HOTEL-MANAGEMENT-SYSTEM

Thanks for your interest! This doc helps you get set up and prepare a pull request that can be merged quickly.

1. Fork the repository and clone your fork.
2. Create a small branch with a clear name, e.g. `fix/typo-readme` or `feat/add-logging`.
3. Keep changes focused and tests-first where possible.
4. Write a clear commit message: `<type>(<area>): short description` — e.g., `feat(readme): add contributing guidelines`.
5. Run the app locally before opening a PR. See `mini-project/README.md` for run instructions.
6. Open a Pull Request and add a brief description and screenshots (if UI changes).

Maintainers: label small, self-contained tasks as `good first issue` and mentor contributors through reviews.
